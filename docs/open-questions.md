# 미결정 사항과 협의 순서

제품 경험이나 구현 범위를 실질적으로 바꾸는 사항만 이 문서에서 협의한다. 안전한
파일 작업, 실제 파일 시스템 우선, 호스트 비종속 Core처럼 명확한 원칙은 별도 선택지
비교 없이 기본 결정으로 채택한다.

## 합의 완료

### Q1. 루트 설정의 최종 소유자

결정: 애플리케이션 설정을 기준으로 사용한다. WPF 네이티브 폴더 선택기가 앱 로컬
설정을 관리한다. Wallpaper Engine/Lively 속성과 시스템 트레이는 사용하지 않는다.
호스트를 바꿔도 기본 루트는 유지된다.

### Q2. 폴더 모달이 표시할 깊이

결정: MVP에서는 실제 폴더의 직접 자식 일반 파일만 모달에 표시한다. 중첩 폴더는
표시하거나 탐색하지 않고 재귀적으로 스캔하지 않는다.

### Q3. 실제 Windows Desktop을 루트로 선택할 때의 아이콘 중복

결정: Wallpaper Engine에서 제공하는 Windows 원본 아이콘 숨김 기능을 사용한다. WPF
애플리케이션은 Explorer 아이콘 상태를 직접 바꾸거나 복원하지 않는다. Lively와
Standalone의 운용 정책은 각 호스트 통합 전에 확정한다.

### 삭제 확인과 충돌 처리

결정: 파일·폴더의 휴지통 이동 전에 Glass 확인 모달을 표시한다. 폴더는 화면에 표시되지
않는 하위 내용도 함께 이동한다는 경고를 표시한다. 영구 삭제는 제공하지 않는다.

내부 파일 이동 중 이름이 충돌하면 `파일명 (n).확장자` 형식의 사용 가능한 이름을
제안하는 모달을 연다. 사용자는 제안 이름을 직접 편집할 수 있으며 기존 파일 덮어쓰기는
제공하지 않는다.

### 모달 개수와 닫기

결정: 한 번에 하나의 모달만 연다. 다른 카드를 클릭하면 내용을 전환하고, 같은 카드를
다시 클릭하거나 모달 바깥 월페이퍼 영역을 좌클릭하거나 `Esc`를 누르면 닫는다.

## 우선순위 B — 상호작용 vertical slice 전에 확정

### Q4. 파일 실행 입력

결정: 한 번 클릭은 선택, 더블클릭은 Windows 기본 연결 프로그램으로 실행한다.

### Q5. 우클릭 메뉴의 MVP 명령

결정: `열기`, `탐색기에서 위치 열기`, `이름 변경`, `휴지통으로 이동`,
`Windows 추가 옵션 표시`를 제공한다. 메뉴 기반 이동은 제공하지 않는다.

`Windows 추가 옵션 표시`는 실제 항목의 네이티브 클래식 Shell 메뉴를 직접 연다. 메뉴가
수행한 변경은 종료 후 재스캔으로 반영한다.

### Q6. Dock 카드 순서

결정: 기본 자연어 이름순으로 표시한다. 사용자가 폴더 카드를 드래그해 재정렬하면 앱
설정에 표시 순서를 보존한다. 카드 재정렬은 실제 파일 시스템을 변경하지 않는다.

가상 `…` 카드는 항상 마지막에 고정하며 왼쪽에 `|` 형태의 얇은 세로 구분선과 적절한
여백을 둔다. 폴더 카드를 다른 카드 안으로 중첩하는 동작은 MVP에서 제외한다.

## 우선순위 C — 호스트 통합 전에 확정

### Q7. 첫 번째 공식 호스트

결정: Standalone으로 WPF 기능을 먼저 검증하고, 첫 공식 호스트는 사용자의 주 사용
환경인 Wallpaper Engine으로 한다. Lively는 그다음 호환 호스트로 개발한다.

Wallpaper Engine 2.8.42부터 Application Wallpaper의 공개 Workshop 배포는 종료됐지만
개인 로컬 실행은 계속 가능하다. 이 프로젝트는 개인 로컬 실행만 대상으로 한다.

결정: Wallpaper Engine/Lively 속성 연동과 시스템 트레이를 사용하지 않는다. 루트 변경은
WPF 내부 설정 패널과 네이티브 폴더 선택기로만 수행한다.

### Q8. 다중 모니터 정책

결정: MVP는 주 모니터 하나만 지원한다. 모니터별 독립 Dock과 여러 모니터를 잇는 surface는
후속 범위로 둔다.

### Q9. 빈 월페이퍼 영역의 우클릭

결정: Dock, 모달과 아이콘이 아닌 순수 월페이퍼 배경을 우클릭하면 디스플레이 설정,
개인 설정과 등록된 배경 확장을 포함하는 Desktop 뷰의 클래식 Shell 메뉴를 표시한다.
파일·폴더 항목의 우클릭은 Glass 메뉴가 소유하고, Dock·모달의 빈 패널 영역은 Windows
메뉴로 전달하지 않는다.

### 아이콘과 미리보기

결정: Dock 폴더 카드는 독립 제작한 macOS풍 아이콘을 사용한다. 일반 파일은 Windows
Shell 연결 아이콘을 사용하고 이미지 파일만 비동기 썸네일을 제공한다. 이미지 이외의
내용 미리보기는 MVP 이후로 미룬다.

### 지원 루트 종류

결정: MVP는 일반 로컬 디렉터리만 지원한다. OneDrive 등 클라우드 동기화 루트,
온라인 전용 파일, 네트워크 드라이브와 UNC 경로는 지원하지 않는다. 디렉터리 심볼릭
링크와 재분석 지점은 따라가지 않는다. OneDrive는 실제 사용 필요가 생길 때 별도로
개발·검증한다.

### 설정 진입 방식

결정: 주 모니터 우측 상단의 보이지 않는 진입 영역을 1초간 연속 hover하면 Glass 설정
패널을 표시한다. 패널 바깥 좌클릭, `Esc`, 닫기 버튼으로 닫는다. 숨은 영역의 우클릭은
Windows 바탕화면 메뉴를 연다. 루트 미설정·손상 상태에서는 설정 패널을 자동 표시한다.

MVP 설정은 루트 표시·변경, 재스캔, 앱 버전과 호스트 상태를 제공한다.

### Dock overflow와 파일 모달

결정: Dock은 주 모니터 작업 영역 하단 중앙에 배치하고 최대 너비를 작업 영역의 80%로
제한한다. 폴더가 많으면 실제 폴더 영역만 가로 스크롤하고, 아이콘 크기와 오른쪽의
구분선·`…` 카드는 유지한다.

파일 모달은 고정 크기 타일을 왼쪽부터 채우며 최대 크기 이후 내부만 세로 스크롤한다.
파일 기본 정렬은 자연어 이름순이다.

### 배경과 데스크톱 UI 레이어

결정: Dock·폴더 모달·시계·영상/음악 정보 카드는 비교적 정적인 Desktop UI Layer로
구성한다. 영상형 배경, 픽셀 애니메이션과 음성 데이터 반응형 2D/3D 이퀄라이저는 그
뒤의 Wallpaper Render Layer에 속한다.

현재 목표에서는 배경 기능을 구현하거나 시각 디자인을 확정하지 않는다. 단, 미래의
동적 렌더러가 WPF UI를 가리거나 같은 FPS로 전체 UI를 재렌더하지 않도록 합성·수명·입력
경계를 지금부터 분리한다.

## 기본 결정 — 별도 협의 없이 적용

- .NET 10 LTS와 WPF를 사용한다.
- 파일 시스템이 유일한 진실의 원천이다.
- Core는 WPF, Win32와 특정 호스트에 의존하지 않는다.
- 파일 작업은 현재 사용자 권한으로만 수행한다.
- 모든 실제 작업은 루트 경계와 재분석 지점을 재검증한다.
- 사용자 확인 없는 덮어쓰기와 영구 삭제를 허용하지 않는다.
- `FileSystemWatcher` 이벤트만 신뢰하지 않고 재스캔으로 복구한다.
- Explorer 등 외부 애플리케이션과의 drag & drop은 MVP에서 제외한다.
- 자동 파일 작업 테스트는 매번 생성한 임시 폴더만 사용한다.
