# 제품 명세

- 상태: 초기 합의안
- 대상 플랫폼: Windows 10/11
- UI 런타임: WPF 네이티브 애플리케이션
- 제품 실행 호스트: Wallpaper Engine
- 개발 표면: 공용 WPF 뷰를 일반 창에 담는 명시적 `--dev-window`
- MVP 디스플레이: 주 모니터 하나
- 배포 성격: 개인용 로컬 애플리케이션, 공개 Workshop 배포 제외

## 1. 제품 목적

감성적인 라이브 월페이퍼가 실제 바탕화면 파일과 폴더 때문에 시각적으로 가려지는
문제를 해결한다. 파일을 없애거나 별도 저장소로 복제하지 않고, 실제 파일 시스템을
macOS Dock에서 영감을 받은 글래스 UI로 다시 구성해 보여준다.

## 2. 핵심 화면 구조

화면 하단에 반투명 Glass & Blur 스타일의 Dock을 배치한다. Dock에는 사용자가 지정한
루트 폴더의 직접 자식 디렉터리와 가상 `…` 카드가 표시된다.

```text
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│                    감성적인 월페이퍼 영역                      │
│                                                             │
│              ┌──────────────────────────┐                   │
│              │ 선택한 카드의 파일 모달     │                   │
│              └──────────────────────────┘                   │
│                                                             │
│             [ 폴더 1 ] [ 폴더 2 ] [ … ]                     │
│                    Glass Dock                               │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 시각 레이어 모델

화면은 성격과 갱신 주기가 다른 레이어로 분리한다.

```text
상단  ┌────────────────────────────────────────────────────┐
      │ Transient Layer                                   │
      │ 설정 · 컨텍스트 메뉴 · drag preview · 확인창       │
      ├────────────────────────────────────────────────────┤
      │ Desktop UI Layer                                  │
      │ Dock · 폴더 모달 · 시계 · 영상/음악 정보 카드       │
      ├────────────────────────────────────────────────────┤
하단  │ Wallpaper Render Layer                            │
      │ 영상 · 픽셀 애니메이션 · 2D/3D 오디오 시각화        │
      └────────────────────────────────────────────────────┘
```

- Wallpaper Render Layer는 화면 전체의 동적 배경을 담당한다.
- Desktop UI Layer는 파일·폴더 UI와 비교적 정적인 위젯을 담당한다.
- Transient Layer는 사용자의 현재 작업과 관련된 가장 높은 우선순위 UI를 담당한다.
- 배경 렌더러의 프레임 갱신이 Dock, 시계와 미디어 카드 전체의 재렌더를 유발하지 않는다.
- 파일 UI의 상태 변경이 배경 애니메이션의 수명과 프레임 루프를 직접 제어하지 않는다.
- Wallpaper Render Layer의 구체 기능은 현재 MVP 파일 UI가 완성된 뒤 개발한다.

현재 단계에서는 Wallpaper Render Layer에 단순한 placeholder만 두고, 영상·픽셀
애니메이션·음성 데이터 반응형 2D/3D 이퀄라이저의 기능과 시각 디자인은 확정하지 않는다.
다만 나중에 추가할 때 Desktop UI Layer를 재구현하지 않도록 합성 경계는 처음부터
유지한다.

## 3. 파일 시스템 투영 규칙

사용자는 하나의 로컬 디렉터리를 루트로 지정한다. 화면 데이터는 별도의 복사본이나
데이터베이스가 아니라 해당 디렉터리의 실제 상태에서 생성한다.

루트 설정의 정본과 변경 UI는 WPF 애플리케이션이 소유한다. 사용자는 앱의 네이티브
폴더 선택기로 루트를 변경하며, Wallpaper Engine 속성이나 시스템 트레이를 설정 경로로
사용하지 않는다. 개발 창도 같은 앱 설정을 사용한다.

예시:

```text
Desktop/
├── Folder1/
├── Folder2/
├── File1.txt
└── File2.png
```

화면 표현:

```text
[ Folder1 ]  [ Folder2 ]  [ … ]
```

| 실제 항목 | 화면 표현 |
|---|---|
| 루트의 직접 자식 디렉터리 | Dock의 실제 폴더 카드 |
| 루트의 직접 자식 일반 파일 | 가상 `…` 카드의 파일 모달 |
| 실제 폴더 카드 클릭 | 해당 폴더의 파일 모달 |
| 가상 `…` 카드 클릭 | 루트 직접 자식 파일 모달 |

`…`는 실제 디렉터리가 아니다. 루트에 직접 속한 파일을 모아 보여주는 UI 전용
컬렉션이며 이름 변경·삭제·파일 시스템 경로를 갖지 않는다.

MVP의 탐색 깊이는 다음으로 제한한다.

- Dock에는 루트의 직접 자식 디렉터리만 표시한다.
- 실제 폴더 모달에는 그 폴더의 직접 자식 일반 파일만 표시한다.
- 폴더 안의 하위 디렉터리와 그 내용은 표시하거나 탐색하지 않는다.
- 하위 디렉터리를 재귀적으로 스캔하지 않는다.

## 4. 진실의 원천과 동기화

실제 파일 시스템이 유일한 진실의 원천이다.

- UI는 파일 시스템을 별도로 복제해 소유하지 않는다.
- 애플리케이션 시작과 루트 변경 시 전체 스캔을 수행한다.
- 실행 중 외부 변경은 `FileSystemWatcher`를 빠른 변경 신호로 사용한다.
- watcher 이벤트 유실과 연속 변경에 대비해 디바운스 후 관련 범위를 다시 스캔한다.
- 파일 작업의 성공은 API 호출 반환만으로 확정하지 않고 재스캔 결과로 확정한다.
- UI와 파일 시스템이 충돌하면 파일 시스템 상태를 우선한다.

## 5. 기본 상호작용

### 5.1 폴더 카드

- 실제 폴더 카드를 클릭하면 해당 폴더 모달을 연다.
- `…` 카드를 클릭하면 루트 파일 모달을 연다.
- 한 번에 하나의 모달만 연다.
- 다른 카드를 클릭하면 열린 모달의 내용을 해당 카드로 전환한다.
- 현재 선택된 카드를 다시 클릭하면 모달을 닫는다.
- 모달 바깥의 월페이퍼 영역을 좌클릭하거나 `Esc`를 누르면 모달을 닫는다.
- 모달은 Dock을 가리지 않고 파일 이동 대상을 계속 선택할 수 있어야 한다.
- 실제 폴더 카드는 기본적으로 자연어 이름순으로 정렬한다.
- 사용자가 폴더 카드를 드래그해 바꾼 표시 순서는 앱 설정에 보존한다.
- 카드 순서 변경은 실제 폴더 이동이 아니며 파일 시스템을 변경하지 않는다.
- 가상 `…` 카드는 항상 마지막에 고정하고, 왼쪽에 세로 구분선과 여백을 두어 실제
  폴더 카드 그룹과 구분한다.
- Dock은 주 모니터 작업 영역 하단 중앙에 배치하고 최대 너비는 작업 영역의 80%로 한다.
- 폴더가 많으면 아이콘을 임의로 축소하지 않고 실제 폴더 영역만 가로 스크롤한다.
- 가로 스크롤 중에도 세로 구분선과 가상 `…` 카드는 Dock 오른쪽에 고정한다.

### 5.2 파일 모달

- 실제 파일을 아이콘과 파일 이름으로 표시한다.
- 일반 파일은 Windows Shell이 제공하는 실제 연결 프로그램 아이콘을 사용한다.
- 이미지 파일은 비동기로 썸네일을 생성하고 로딩 중에는 Shell 아이콘을 표시한다.
- 이미지 외 형식의 내용 미리보기는 MVP에서 제공하지 않는다.
- 파일 타일은 고정 크기 격자로 왼쪽부터 채운다.
- 모달은 파일 수에 따라 계속 커지지 않고 작업 영역 안의 최대 크기를 유지한다.
- 항목이 최대 영역을 넘으면 모달 내부만 세로 스크롤한다.
- 파일 기본 정렬은 자연어 이름순이다.
- 한 번 클릭은 파일을 선택하고, 더블클릭은 Windows 기본 연결 프로그램으로 실행한다.
- 파일 선택, 실행, 우클릭 메뉴와 드래그 시작점을 제공한다.
- 표시 항목 수가 많을 때 Dock과 월페이퍼의 안전 영역을 침범하지 않아야 한다.

### 5.3 내부 드래그 앤 드롭

MVP는 애플리케이션 내부 파일 이동을 지원한다.

| 출발 | 도착 | 실제 결과 |
|---|---|---|
| `…`의 파일 | 실제 폴더 카드 | 루트에서 해당 폴더로 파일 이동 |
| 폴더 A의 파일 | 폴더 B 카드 | 폴더 A에서 폴더 B로 파일 이동 |
| 실제 폴더의 파일 | `…` 카드 | 해당 폴더에서 루트로 파일 이동 |

드래그 중에는 유효한 대상, 무효한 대상과 이동 예정 상태를 명확히 표시한다. 이름 충돌,
권한 부족, 파일 잠금 또는 외부 변경이 발생하면 실제 상태를 바꾸지 않은 채 오류를
표시하거나 재스캔 결과로 복구한다. 사용자 확인 없이 같은 이름의 파일을 덮어쓰지 않는다.

대상 위치에 같은 이름이 있으면 확장자를 보존한 `파일명 (n).확장자` 형식으로 가장 작은
사용 가능한 양의 정수 `n`을 계산한다. 충돌 모달은 이 이름을 입력란의 기본값으로
제안하며 사용자가 이동 전에 직접 수정할 수 있다. 최종 입력 이름은 Windows 파일 이름
규칙, 대상 충돌과 루트 경계를 다시 검증한다. MVP에서는 기존 파일 덮어쓰기를 제공하지
않는다.

### 5.4 우클릭

파일과 폴더 카드의 우클릭은 WPF 애플리케이션이 직접 처리한다. 항목 위에서 발생한
우클릭을 Windows 바탕화면 또는 실행 호스트가 대신 처리해서는 안 된다. 메뉴는
Glass 스타일로 애플리케이션이 렌더링한다.

MVP 커스텀 메뉴는 다음 명령을 제공한다.

1. `열기`
2. `탐색기에서 위치 열기`
3. `이름 변경`
4. `휴지통으로 이동`
5. `Windows 추가 옵션 표시`

`다른 카드로 이동` 명령은 메뉴에 두지 않는다. 카드 사이 이동은 내부 drag & drop으로
수행한다.

`휴지통으로 이동`은 Windows Shell의 recycle 동작을 사용하며 영구 삭제로 대체하지
않는다. `Windows 추가 옵션 표시`는 현재 항목을 다시 검증한 뒤 해당 파일 또는 폴더의
실제 클래식 Shell 메뉴를 직접 연다. 메뉴는 현재 커서가 아니라 최초 항목
우클릭의 화면 위치를 기준으로 표시한다. 실행한 명령은 Windows Shell이 소유하며, 메뉴가 닫힌 뒤
애플리케이션은 실제 파일 시스템을 다시 스캔한다.

휴지통 이동 전에는 Glass 확인 모달을 표시한다. 폴더인 경우 현재 화면에 표시되지 않는
하위 폴더와 파일도 함께 휴지통으로 이동한다는 사실을 명시한다.

### 5.5 빈 월페이퍼 영역의 우클릭

Dock, 폴더 모달, 파일·폴더 아이콘이 아닌 순수 월페이퍼 배경을 우클릭하면 Desktop
Shell 뷰가 제공하는 실제 클래식 컨텍스트 메뉴를 표시한다. 디스플레이 설정, 개인 설정과
설치된 NVIDIA 제어판 같은 배경 Shell 확장을 Windows가 제공하는 그대로 사용할 수 있어야 한다.

| 우클릭 위치 | 동작 |
|---|---|
| 파일 또는 실제 폴더 카드 | Glass 커스텀 메뉴 |
| 순수 월페이퍼 배경 | Windows 바탕화면 배경 메뉴 |
| Dock 또는 모달의 비어 있는 패널 영역 | Windows 메뉴로 전달하지 않음 |

배경 메뉴를 열 때 기존 Glass 메뉴 또는 폴더 모달이 열려 있으면 먼저 닫는다. 메뉴
명령을 앱에 정적으로 복제하지 않고 Windows Shell이 만든 네이티브 메뉴를 직접 표시한다.

### 5.6 숨겨진 설정 패널

설정 UI는 항상 노출하지 않는다. 주 모니터 우측 상단에 시각적으로 보이지 않는 작은
진입 영역을 두고, 포인터가 해당 영역에 1초 동안 연속으로 머물면 Glass 설정 패널을
fade/slide 방식으로 표시한다.

- 1초가 되기 전에 포인터가 영역을 벗어나면 열기 타이머를 취소한다.
- 열린 설정 패널로 포인터를 이동해도 패널이 닫히지 않는다.
- 패널 바깥 좌클릭, `Esc` 또는 명시적인 닫기 버튼으로 닫는다.
- 설정 패널이 열린 동안 폴더 모달과 항목 컨텍스트 메뉴는 닫는다.
- 숨은 진입 영역의 우클릭은 소비하지 않고 Windows 바탕화면 메뉴를 연다.
- 움직임 줄이기가 활성화된 경우 fade/slide를 생략하고 즉시 표시한다.

MVP 설정 패널은 다음 기능을 제공한다.

- 현재 루트 표시
- 네이티브 폴더 선택기를 통한 루트 변경
- 파일 시스템 다시 스캔
- 현재 앱 버전과 실행 호스트 상태 확인

루트가 아직 선택되지 않았거나 더 이상 유효하지 않으면 숨은 진입 방식만 강요하지
않고 시작 화면에서 설정 패널을 자동으로 표시한다.

## 6. 시각 디자인

macOS의 시각 언어에서 영감을 받은 Glass & Blur 스타일을 사용하되 macOS 자산을
복제하지 않고 독립적인 디자인으로 구현한다.

- 반투명 Dock과 폴더 모달
- 배경과 분리되는 블러, 얇은 테두리와 절제된 그림자
- 둥근 모서리와 아이콘 중심의 간결한 정보 구조
- 실제 폴더 그룹과 가상 `…` 카드 사이의 얇은 세로 구분선
- Dock의 실제 폴더 카드는 독립 제작한 macOS풍 폴더 아이콘 사용
- hover, 선택, drag, drop, 오류 상태의 일관된 애니메이션
- 월페이퍼를 가리지 않으면서 텍스트와 아이콘 대비를 유지하는 색상 체계
- 움직임 줄이기와 Windows DPI 배율을 고려한 표현

## 7. MVP 범위에서 제외

- Windows Explorer와 애플리케이션 사이의 외부 드래그 앤 드롭
- 다른 파일 관리자 또는 애플리케이션과의 OLE 드래그 앤 드롭
- 루트 경계 밖 임의 파일 탐색
- 관리자 권한이 필요한 파일 작업
- 사용자 확인 없는 덮어쓰기
- 호스트별 웹 UI 또는 브라우저 샌드박스에 핵심 기능을 의존하는 구조
- 폴더 모달 안의 하위 디렉터리 탐색
- 보조 모니터 또는 모니터별 독립 Dock
- OneDrive 등 클라우드 동기화 루트와 온라인 전용 파일
- 네트워크 드라이브와 UNC 경로
- 이미지 외 파일 형식의 내용 미리보기

## 8. Windows 원본 바탕화면 아이콘

WPF 애플리케이션은 Windows Explorer의 원본 바탕화면 아이콘 표시 상태를 직접
변경하거나 복원하지 않는다. Wallpaper Engine에서 제공하는 원본 아이콘 숨김 기능을
사용자가 활성화하는 것을 기본 운용 방식으로 한다.

이 정책은 아이콘 상태의 소유권을 Wallpaper Engine에 유지하고, 애플리케이션 비정상
종료가 Explorer 아이콘 상태를 잘못 남기는 문제를 방지한다. 개발 창은 제품 운용 경로가
아니므로 Explorer 아이콘을 변경하지 않는다.

## 9. 안전 원칙

- 모든 파일 작업 직전에 원본과 대상이 현재 루트 경계 안인지 다시 확인한다.
- 상대 경로 탈출, 심볼릭 링크와 재분석 지점을 통한 경계 우회를 허용하지 않는다.
- MVP 루트는 로컬 파일 시스템의 일반 디렉터리만 허용한다.
- 심볼릭 링크와 디렉터리 재분석 지점은 표시·탐색·작업 대상으로 사용하지 않는다.
- 셸 명령 문자열을 조합하지 않고 .NET 및 Windows Shell API를 사용한다.
- 관리자 권한 없이 현재 사용자 권한으로 실행한다.
- 영구 삭제를 MVP의 기본 동작으로 제공하지 않는다.
- 실패한 파일 작업을 UI에서 성공한 것처럼 확정하지 않는다.
