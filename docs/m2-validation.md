# M2 검증 기록

- 상태: 통과
- 검증일: 2026-07-20
- Windows: 11, build 26200
- .NET SDK: 10.0.302

## 자동 검증

Linux와 Windows 로컬 파일 시스템에서 각각 Release restore/build/test를 실행했다.

| 범위 | 결과 |
|---|---|
| Core 테스트 | 21 통과 |
| Windows infrastructure 테스트 | 5 통과 |
| WPF Release 빌드 | 경고 0, 오류 0 |
| watcher 반복 실행 | 4회 연속 통과 |

자동 테스트에는 자연어 정렬, 빈 루트·빈 폴더, content 접근이 잠긴 파일, 1,500개 파일
snapshot, 저장 순서 병합·이동, watcher debounce, 실제 외부 변경 신호, 루트 삭제와
재생성 감지가 포함된다.

## Windows 실제 UI 검증

`new-m2-fixture.ps1`의 로컬 임시 fixture와 Standalone WPF 창에서 다음을 확인했다.

| 요구사항 | 검증 결과 |
|---|---|
| Windows Shell 아이콘 | `.md`와 `.txt`가 현재 Windows 연결에 따른 서로 다른 실제 아이콘으로 표시됨 |
| 이미지 썸네일·캐시 | PNG가 비동기 썸네일로 전환되고 외부 이미지 갱신 후 새 cache key로 즉시 교체됨 |
| watcher·debounce·재스캔 | 열린 모달에서 외부 생성 파일이 자동 표시되고 연속 이벤트는 단일 신호로 합쳐짐 |
| 루트 소실·복구 | 루트 소실 시 snapshot과 Dock을 비우고 설정 패널을 열며, 같은 경로 복구 시 자동 재구성됨 |
| Dock 재정렬·저장 | drag 삽입선 표시, 순서 변경, `settings.json` 저장과 프로세스 재시작 후 복원 확인 |
| 빈 폴더 | 빈 상태 메시지와 파일 수 0 상태가 표시됨 |
| 다량 파일 | 1,200개 모달에서 20개 타일만 실체화되고 마지막 파일까지 스크롤 가능하며 UI 응답 유지 |
| 권한 오류 | 접근 거부 하위 폴더를 제외하고 `접근할 수 없는 항목 1개` 상태로 계속 동작 |

검증은 실제 사용자 파일이 아닌 매번 생성한 로컬 fixture만 사용했다. M2는 읽기 전용이며
이 검증 중 파일 생성·ACL 변경·루트 소실은 fixture 안에서만 수행하고 원래 상태로 복구했다.
