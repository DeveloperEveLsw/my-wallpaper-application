# ADR 0002: 앱 소유 루트 설정과 얕은 파일 시스템 투영

- 상태: 채택
- 날짜: 2026-07-19

## 배경

애플리케이션은 Wallpaper Engine, Lively와 Standalone 실행을 지원해야 한다. 호스트마다
루트 설정을 따로 소유하면 같은 앱이 호스트에 따라 다른 파일을 보여주고 설정 이전과
문제 재현이 어려워진다. 또한 재귀 폴더 탐색은 초기 제품을 범용 파일 관리자로 확장한다.

## 결정

- 루트 설정의 정본은 WPF 애플리케이션이 소유한다.
- 루트 변경은 WPF 내부 설정 패널과 네이티브 폴더 선택기로만 수행한다.
- 호스트 속성과 시스템 트레이는 설정 경로로 사용하지 않는다.
- Dock은 루트의 직접 자식 디렉터리를 표시한다.
- 가상 `…` 카드는 루트의 직접 자식 일반 파일을 표시한다.
- 실제 폴더 모달은 해당 폴더의 직접 자식 일반 파일만 표시한다.
- 중첩 디렉터리는 MVP에서 표시·탐색·재귀 스캔하지 않는다.
- Wallpaper Engine 운용 시 원본 Explorer 아이콘은 Wallpaper Engine 기능으로 숨긴다.
- WPF 앱은 Explorer 아이콘 표시 상태를 직접 소유하지 않는다.

## 결과

- 호스트 교체 후에도 같은 루트 설정을 유지할 수 있다.
- 파일 시스템 스캔과 경계 검증의 깊이가 명확해진다.
- 내부 파일 이동의 출발지와 대상이 루트 및 직접 자식 폴더로 제한된다.
- Windows 원본 아이콘 상태 변경 실패와 비정상 종료 복구 책임이 WPF 앱에서 제거된다.
- Lively와 Standalone의 원본 아이콘 운용 방식은 아직 별도 결정이 필요하다.
