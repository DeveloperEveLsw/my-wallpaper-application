# ADR 0005: 우측 상단 hover로 여는 WPF 설정 패널

- 상태: 채택
- 날짜: 2026-07-19

## 결정

- 설정은 WPF 애플리케이션 내부 Glass 패널로 제공한다.
- Wallpaper Engine/Lively 속성과 시스템 트레이는 설정 진입점으로 사용하지 않는다.
- 주 모니터 우측 상단의 보이지 않는 진입 영역을 1초간 연속 hover하면 패널을 표시한다.
- 패널 바깥 좌클릭, `Esc` 또는 닫기 버튼으로 패널을 닫는다.
- 진입 영역의 우클릭은 Windows 바탕화면 배경 메뉴를 연다.
- 루트 미설정 또는 설정 손상 상태에서는 패널을 자동 표시한다.

MVP 패널은 루트 표시·변경, 수동 재스캔, 앱 버전과 현재 호스트 상태를 제공한다.
폴더 선택은 WPF 프로세스가 연 Windows 네이티브 폴더 선택기를 사용한다.

## 이유

설정 버튼을 상시 노출하지 않아 감성적인 월페이퍼를 보존하면서도 별도 트레이 UI나
호스트별 속성 계약 없이 설정에 접근할 수 있다. 1초 dwell은 우연히 모서리를 스치는
동작과 의도적인 접근을 구분한다.

## 안전장치

- hover 타이머는 영역 이탈 즉시 취소한다.
- 설정 패널이 열린 뒤에는 패널로 이동하는 포인터를 영역 이탈로 처리하지 않는다.
- 루트가 없을 때 숨은 진입점만 제공하지 않는다.
- reduced motion에서는 전환 애니메이션을 생략한다.
- 입력 전달 실패를 진단할 수 있도록 개발 모드에는 직접 설정 열기 인자를 제공할 수 있다.
