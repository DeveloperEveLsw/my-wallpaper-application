# ADR 0004: 주 모니터·로컬 루트와 아이콘 범위

- 상태: 채택
- 날짜: 2026-07-19

## 결정

- MVP는 주 모니터 하나만 지원한다.
- 루트는 일반 로컬 파일 시스템 디렉터리만 허용한다.
- 네트워크·UNC·OneDrive 및 기타 클라우드 동기화 루트는 지원하지 않는다.
- 심볼릭 링크와 디렉터리 재분석 지점은 표시·탐색·작업 대상으로 사용하지 않는다.
- Dock의 실제 폴더 카드는 독립 제작한 macOS풍 폴더 아이콘을 사용한다.
- 일반 파일은 Windows Shell 연결 아이콘을 표시한다.
- 이미지 파일은 비동기 썸네일을 표시하고 로딩·실패 중에는 Shell 아이콘으로 폴백한다.
- 이미지 이외의 파일 내용 미리보기는 MVP에서 제외한다.
- 호스트 구현은 Standalone, Wallpaper Engine 로컬 실행, Lively 순서로 진행한다.

## 이유

초기 사용 환경에서 OneDrive와 다중 모니터가 필요하지 않다. 클라우드 placeholder,
재분석 지점과 모니터별 호스트 생명주기를 미루면 실제 사용하는 단일 환경의 핵심 파일
동작과 입력을 더 빠르게 검증할 수 있다.

파일 연결 아이콘은 사용자가 Windows에서 익숙한 파일 형식을 유지한다. 폴더 카드만
제품 고유의 시각 언어를 사용하고, 이미지 썸네일은 감성적인 모달 경험을 보강한다.

## 후속 조건

OneDrive 또는 다중 모니터가 실제로 필요해지면 placeholder hydration, Known Folder
redirect, 재분석 지점, 모니터별 DPI·work area와 호스트 pause/resume을 별도 단계로
검증한 뒤 지원한다.
