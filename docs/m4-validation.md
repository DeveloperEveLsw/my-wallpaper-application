# M4 검수 체크리스트

- 구현 상태: 완료
- 자동 검증: 통과
- Windows 세부 UI 검수: 사용자 진행 예정
- 기준일: 2026-07-21

## 구현된 범위

- `…`의 루트 파일을 실제 폴더 카드로 이동
- 실제 폴더 파일을 `…`의 루트 위치로 이동
- 실제 폴더 사이의 파일 이동
- Dock 순서 drag와 구분된 내부 파일 drag payload
- 포인터를 따라가는 파일명 drag preview
- 현재 출발 카드의 무효 표시와 이동 가능한 카드의 유효 표시
- 파일 이동 직전 출발 파일·루트·목적 폴더·깊이·재분석 지점 재검증
- 대상 위치의 파일과 표시되지 않는 하위 폴더를 모두 포함한 대소문자 무시 이름 충돌 검사
- 확장자를 보존한 가장 작은 `파일명 (n).ext` 제안과 편집 가능한 Glass 모달
- 사용자 확인 없는 덮어쓰기 금지
- 성공·실패·drag 취소·충돌 모달 취소·외부 변경 뒤 전체 snapshot 재스캔

M4는 단일 파일의 애플리케이션 내부 이동만 제공한다. 폴더 이동, 복사, 다중 선택,
실행 취소, Explorer 등 외부 애플리케이션과의 drag & drop은 범위 밖이다.

## 자동 검증 범위

Release build와 전체 테스트에서 다음 항목을 확인한다.

- 루트 → 폴더, 폴더 → 루트, 폴더 → 폴더 이동과 파일 내용 보존
- 루트 또는 루트의 직접 자식 폴더만 목적지로 허용
- 같은 카드, 중첩 목적지, 사라진 출발 파일과 사라진 목적 폴더 거부
- 폴더를 이동 출발 항목으로 사용하는 명령 거부
- Windows에서 사용할 수 없는 편집 이름 거부
- 파일 및 화면에 표시되지 않는 폴더와의 이름 충돌 감지
- 대소문자를 무시한 충돌과 가장 작은 사용 가능 양의 정수 접미사 선택
- 확장자 없는 이름, 다중 확장자, 점으로 시작하는 이름과 255자 이름 제안
- 동일 이름 충돌에서 출발 파일과 기존 대상 내용을 모두 보존
- 이동 준비 뒤 외부에서 생긴 동시 이름 충돌도 실행 직전에 다시 거부
- 잠긴 Windows 파일 이동 실패에서 출발 파일 보존
- WPF Release 빌드 경고 0, 오류 0

## 사용자 검수 준비

Windows PowerShell에서 다음을 실행한다.

```powershell
./scripts/check.ps1
$fixture = ./scripts/new-m4-fixture.ps1
./scripts/run-standalone.ps1 -RootPath $fixture.RootPath -Configuration Release
```

### drag preview와 대상 상태

- `Work` 카드를 열고 `work-to-archive.txt`를 누른 채 움직이면 파일명 preview가 포인터를 따라오는지 확인한다.
- 같은 `Work` 카드 위에서는 붉은 `이동 불가` 표시와 금지 커서가 나오는지 확인한다.
- `Archive`, `Photos`, `…` 위에서는 초록 이동 표시와 이동 커서가 나오는지 확인한다.
- Dock의 카드가 아닌 배경에 놓거나 `Esc`로 drag를 취소하면 파일이 원래 위치에 남는지 확인한다.

### 세 이동 경로

- `…`의 `root-to-work.txt`를 `Work`로 이동하고 실제 파일 경로와 열린 snapshot이 갱신되는지 확인한다.
- `Work`의 `work-to-root.txt`를 `…`로 이동하고 루트 파일 모달에 나타나는지 확인한다.
- `Work`의 `work-to-archive.txt`를 `Archive`로 이동하고 파일 내용이 보존되는지 확인한다.
- 성공 알림 전후에 watcher 재스캔이 중복 항목이나 일시적인 유령 타일을 만들지 않는지 확인한다.

### 이름 충돌과 편집 모달

- `Work/collision.txt`를 `Archive`로 끌면 `collision (3).txt`가 제안되는지 확인한다.
  `Archive`에는 파일 `collision.txt`, `collision (1).txt`와 표시되지 않는 폴더
  `collision (2).txt`가 의도적으로 존재한다.
- 제안 이름을 `moved-collision.txt`로 편집해 이동하고 기존 세 항목이 모두 보존되는지 확인한다.
- 다시 fixture를 만든 뒤 충돌 모달에서 `CON.txt`, `bad?.txt`, 끝 공백·마침표를 입력하면 이동 버튼이 비활성화되는지 확인한다.
- 충돌 모달에서 취소하면 출발 파일과 모든 대상 항목이 그대로 남고 snapshot이 갱신되는지 확인한다.
- 충돌 모달이 열린 동안 Explorer에서 제안 이름을 새로 만든 뒤 확정하면 덮어쓰지 않고 새 제안을 표시하는지 확인한다.

### 잠금과 외부 변경 복구

다음 PowerShell로 파일을 잠근 상태에서 `locked-move.txt`를 다른 카드로 이동한다.

```powershell
$lock = [System.IO.File]::Open($fixture.LockedFilePath, 'Open', 'ReadWrite', 'None')
# 앱에서 이동을 시도한 뒤
$lock.Dispose()
```

- 잠금 오류가 표시되고 출발 파일과 대상 위치가 변경되지 않는지 확인한다.
- `stale-source.txt`를 `Archive`로 drag해 충돌 모달을 연 상태에서 출발 파일을 Explorer로
  삭제하거나 이름을 바꾼 뒤 확정하면 stale 오류와 최신 snapshot이 표시되는지 확인한다.
- 목적 폴더를 외부에서 이름 변경·삭제한 뒤 확정해도 루트 밖이나 사라진 위치로 이동하지 않는지 확인한다.

### M2·M3 회귀

- 폴더 카드 자체를 drag하면 파일 이동이 아니라 기존 Dock 순서 재정렬이 수행되는지 확인한다.
- Dock 순서를 저장한 뒤 앱을 재시작해 순서가 유지되는지 확인한다.
- 파일 선택·더블클릭 열기와 파일·폴더 우클릭 메뉴가 정상 동작하는지 확인한다.
- 이름 변경과 휴지통 이동, Shell 아이콘·이미지 썸네일, 대량 파일 가상화가 유지되는지 확인한다.
